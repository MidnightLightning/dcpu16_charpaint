This is a visual character creator for the DCPU-16, a fictitious, virtual CPU used in the upcoming 0x10c game by Notch. The DCPU-16 has a default video memory address space, and an address space for defining the shape of the characters drawn on the screen. There is a default alphabet defined (like this letter 'A'), but written programs can redefine them to create custom characters, and in the future there may be the option for sprite sheets and more advanced control of the individual pixels of the screen.

To use, draw the character you wish to represent in the grid above (gray is off, orange is on. Click a square to toggle its state. Click and drag to "paint" with that state). Then take the hexadecimal output (two "words" in DCPU-16 assembly) and put them the proper location to be accessed for video RAM use (address 0x8180 is character 0, 0x8182 is character 1, all the way to 0x8280.)

Sample code (generates character drawn above; try it out on 0x10co.de):

Customize: Glyph slot: Foreground color: Background color: 

; Create custom character in slot 00
set [0x8180], 0x0354
set [0x8181], 0xa403

set [0x8000], 0xf100 ; Character 00, White foreground, blue background

Note that this feature is not part of the DCPU-16 official specification yet! It has been ascertained that this functionality will be available due to leaked JAR files, but may change quickly. This page will strive to keep up with the official specification, when it's officially released.
